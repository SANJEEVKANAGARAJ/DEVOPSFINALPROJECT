Docker & Containerization

Understanding how we package and run our software reliably.

What is Docker?

Docker is a platform for developing, shipping, and running applications in containers. A container is a lightweight, standalone, executable package that includes everything needed to run an application: code, runtime, system tools, system libraries, and settings.

Why use Containers?

Containers solve the "it works on my machine" problem. They ensure that your application works the same in development, testing, and production by bundling the environment along with the code.

Containers vs Virtual Machines

Feature Containers Virtual Machines
Guest OS Shares Host OS kernel Every VM has its own OS
Startup Seconds (Near-instant) Minutes (Heavy boot)
Size MBs (Very light) GBs (Very heavy)
Performance Native speed Overhead due to Hypervisor

Our Dockerfile Explained

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image.

FROM nginx:alpine # Use lightweight Nginx
COPY frontend/ /usr/share/nginx/html/ # Copy code
EXPOSE 80 # Open web port
CMD ["nginx", "-g", "daemon off;"] # Start server
  • FROM: Sets the base image. We use nginx:alpine because it is extremely small (~5MB), secure, and optimized for speed.
  • COPY: Moves our static HTML files from the local machine into the Docker image's web root directory so Nginx can serve them.
  • EXPOSE: Informs Docker that the container listens on port 80 at runtime.
  • CMD: The command that runs when the container starts. Here, it starts the Nginx server in the foreground.

How Build Works

docker build -t my-app .

Docker reads the file, creates layers for each instruction, and caches them for faster subsequent builds. The result is an Image.

How Run Works

docker run -p 80:80 my-app

Docker creates a Container (a running instance) from the image. It maps the host port to the container port.
